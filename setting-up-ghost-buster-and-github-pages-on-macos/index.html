Setting up Ghost, Buster and GitHub Pages on MacOS [Yosemite]

01 September 2015 on Ghost, Git

In the last 5-8 years, I am not sure for how many times did I try to start to blog. It was always something in my way; not sure what to write about, not fully satisfied with blogging platform, editor sucked, deployment was a pain, etc.

In short it was just a lack of my interest.

Now, when I am going through a lot of changes both in my professional and personal life, there's a publisher's spark in me that feels right. So I took one day to do a research on blog engines and static site generators, and it was not that hard to do it.

I've found a setup that fits me quite well. It is combined out of 3 components:

  • GitHub Pages
  • Ghost blogging platform
  • Buster static content generator

If you just want to see the setup, feel free to skip the introductory text.

GitHub Pages

As a web developer I honestly support open source community, and what is a better place than GitHub. GitHub provides a service called GitHub Pages, which enables you or your project a place to host a static web content (website or a blog).

From developer's perspective this approach provides multiple benefits. For example, when you apply for new job, companies will often ask you your GitHub account so they can check your coding style and skill level. Not so often (it happened to me twice so far) they'll ask about your blog, interested in your way of thinking and industry savviness. By using GitHub Pages you have it all in one place.

Learn more on GitHub Pages official website.

Ghost

Now, GitHub Pages website promotes Jekyll which is really powerful static site generator, but most of it's plugins won't work inside of GitHub Pages. One of the main things I was paying attention while deciding on which blogging platform to use was workflow.

And Ghost came as the best choice. Key features that made up my mind were:

  • editor - fast, beautiful and to the point (when I saw it, I knew I want to use it)
  • the whole application - nothing is in your way and all you need is there
  • themes - no need to spend time on internals like tag filtering or pagination and RSS

Ghost is Node.js based application and child of John O'Nolan, former deputy head of the Wordpress UI group, so no wonder that John and his team have done a great job!

But... :) .... Ghost is not static content generator (remember GitHub Pages serves only static content).

So, who you gonna call?

For more detailed information on Ghost, please take a look at official user guide.

Buster

Buster is Python package, developed by Akshit Khurana, that serves as static content generator for Ghost. In my setup I am using Josh Gerdes's fork as, at the time of writing this article, his repo is 4 commits ahead of Akshit's, containing some important fixes.

Setup

Before we start, we have to meet some requirements.

Ghost is built using Node.js and it will need npm package manager for installation and server startup.

Buster is Python package, so we'll use PIP. Python comes pre-installed on MacOs systems and for PIP follow this how to guide.

Git should also be available on your system, but if it's not use Homebrew to install it or do it in your prefered way.

Also you will need to install latest XCode for some libraries, unless you already have it.

Requirements

  1. Let's start with Node.js and npm. You can download latest package from http://nodejs.org/ then run it and setup $PATH variable or you can just use Homebrew (like me) and type:

    brew install node
